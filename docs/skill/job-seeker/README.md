This directory is a **portable export** of the `job-seeker` (申职器) skill. The content is editor-agnostic; pick the import path that matches your environment.

## Import paths

### WorkBuddy (macOS / Windows)

WorkBuddy loads skills from `~/.workbuddy/skills/` (user-level) or `<workspace>/.workbuddy/skills/` (project-level).

**Project-level (recommended — travels with the repo):**

```bash
mkdir -p .workbuddy/skills/
cp -r docs/skill/job-seeker .workbuddy/skills/
# Add agent_created: true to .workbuddy/skills/job-seeker/SKILL.md frontmatter
# so SkillManage can modify it later.
```

Then either restart WorkBuddy or run the skill-creator's `quick_validate.py` to confirm it loads.

**User-level (available in every project):**

```bash
cp -r docs/skill/job-seeker ~/.workbuddy/skills/
```

### Cursor

Cursor loads skills from `.cursor/skills/` (project-level).

```bash
mkdir -p .cursor/skills/
cp -r docs/skill/job-seeker .cursor/skills/
# Cursor ignores the agent_created field — no edit needed.
```

### Generic editor / manual use

Just open `SKILL.md` in any editor and follow the workflow by hand. The build script (`./build_resumes.sh` from the repo root) does the heavy lifting; the skill is mainly documentation and checklists.

## Validation

If you have the WorkBuddy `skill-creator` builtin available, validate the imported skill:

```bash
python /Applications/WorkBuddy.app/Contents/Resources/app.asar.unpacked/resources/builtin-skills/skill-creator/quick_validate.py .workbuddy/skills/job-seeker
```

Expected: `OK` with no errors. The `name` field must match the directory name (`job-seeker`) and contain only `[a-z0-9-]`.

## Maintenance

- The canonical source is `.workbuddy/skills/job-seeker/` (with `agent_created: true`).
- This `docs/skill/job-seeker/` mirror is regenerated by copying the canonical and stripping `agent_created: true`.
- When updating the skill, edit the canonical first, then re-export to this mirror.

## Placeholders

All example content in this skill uses placeholders (`YOUR_NAME`, `your-email@example.com`, `your-org/job-seeker`, `YOUR_OPEN_SOURCE_PROJECT`). Do **not** replace these with real maintainer identifiers in the public/open-source copy. Replace them only in your local private fork.
